# Phase 2 — cancelled

These are the documents for a successor project that was **cancelled at the spec stage**,
before any application code was written. They are kept here because the reasoning is the
point: this is what was planned, and why the evidence stopped it.

- [`2026-07-19-exposure-ledger-design.md`](2026-07-19-exposure-ledger-design.md) — the
  design spec, carrying a "partially superseded" header noting what became stale.
- [`INHERITED-FROM-V1.md`](INHERITED-FROM-V1.md) — what phase 2 would have taken from this
  codebase, what it would have left, and the lessons it was meant to avoid repeating.

Why it was cancelled is set out in [`../CASE-STUDY.md`](../CASE-STUDY.md) §4–5. The short
version: the planned headline feature — a personal inhaled-dose ledger — sits in the
product category with the worst documented retention (70% abandonment within 100 days),
depends on a daily habit the literature says does not form, and its most novel feature
turned out to be mathematically unsupportable by the available data.

The originating repository (`~/Projects/saafsaans`) contains only these documents and a
README. No application code was ever written.
